#  Android 4대 컴포넌트 및 구성 요소

**컴포넌트**는 소프트웨어 시스템을 구성하는 **독립적이고 재사용 가능한 모듈** 또는 구성 요소를 의미합니다.

안드로이드에서 **컴포넌트**란, 안드로이드 시스템이 직접 실행하고 생명주기를 제어하는 독립적인 실행 단위를 의미합니다.



---

##  안드로이드 4대 핵심 컴포넌트

4대 컴포넌트(Activity, Service, Broadcast Receiver, Content Provider)는 앱의 **진입점**으로서

안드로이드 OS가 직접 생성, 중지, 재시작을 관리하는 핵심 실행 단위이기 때문에 특별하게 분류됩니다.-

### 1. Activity (액티비티) 
* **정의:** 사용자와 상호작용을 담당하는 **인터페이스 (UI 화면)** 입니다.
* **특징:**
    * 앱 실행 시 앱 전체가 아닌 특정 **액티비티를 호출**하여 시작됩니다.
    * **생명주기** 관련 메서드들을 재정의하여 원하는 기능을 구현할 수 있습니다.
    * 액티비티는 인텐트(Intent)를 통해 다른 Application의 액티비티를 호출할 수 있습니다.
    * **2개 이상**의 액티비티를 동시에 Display 할 수 없습니다.
    * **1개 이상**의 View(텍스트, 버튼, 이미지) 또는 ViewGroup(레이아웃)을 포함합니다.
    * 액티비티 내에 **프래그먼트**를 추가하여 화면을 분할시킬 수 있습니다.

### 2. Service (서비스) 
* **정의:** 사용자와 직접적으로 상호작용하지 않으며, **백그라운드**에서 작업을 처리하기 위해 주로 사용됩니다.
* **특징:**
    * Application이 종료되어도 BackGround에서 동작하는 컴포넌트입니다. (예: 음악 재생, 타이머)
    * **액티비티와 동일한 UI 스레드**로 실행됩니다.
    * **포그라운드 서비스:** 사용자에게 잘 보이는 작업으로, 반드시 **알림을 표시**해야 합니다.
    * **백그라운드 서비스:** 사용자에게 직접 보이지 않는 작업.
    * **바인드 서비스:** 다른 앱 컴포넌트가 `bindService()`를 호출해 서비스를 호출하면 서비스가 바인딩되며, 클라이언트-서버 인터페이스를 제공하여 상호 작용합니다. 바인딩된 컴포넌트가 모두 종료되면 서비스도 종료됩니다.

### 3. Broadcast Receiver (방송 수신자) 
* **정의:** 안드로이드 OS로부터 발생하는 각종 **이벤트와 정보**를 받아 핸들링하는 컴포넌트입니다.
* **특징:**
    * 안드로이드 디바이스의 특수한 상황에 대응합니다. (예: 시스템 부팅 완료, 네트워크 연결/끊김, 배터리 부족 알림)
    * 거의 대부분 **UI를 가지지 않습니다.**
    * 수신기를 통해 디바이스 상황을 감시하다가 이벤트가 발생하면 해당 이벤트에 맞게 정의한 작업을 수행합니다.
    * 특정한 상황을 제외하고는 브로드캐스트는 **시스템에서 시작**됩니다.

### 4. Content Provider (콘텐트 제공자) 
* **정의:** 데이터를 관리하고 **다른 Application의 데이터를 제공**하는데 사용되는 컴포넌트입니다.
* **특징:**
    * **애플리케이션 간의 데이터 공유**를 위해 표준화된 인터페이스를 제공합니다.
    * 외부 어플리케이션이 현재 앱의 데이터베이스에 함부로 접근하지 못하게 하면서, **공개하고 공유하고 싶은 데이터만** 공유할 수 있도록 돕습니다.
    * SQLite DB, Web, 파일 입출력 등을 통해 데이터를 관리합니다.
    * 용량이 큰 데이터(음악, 사진 등) 공유에 적합합니다. (작은 데이터는 Intent 사용)
    * 데이터의 Read, Write에 대한 **퍼미션**이 있어야 접근이 가능합니다.
    * 데이터베이스에서 흔히 사용되는 **CRUD** (Create, Read, Update, Delete) 원칙을 준수합니다.


### 특징  

1. 4대 컴포넌트의 가장 큰 특징은 개발자가 new 키워드로 객체를 직접 생성하지 않는다는 점입니다. 안드로이드 시스템이 AndroidManifest.xml을 참고하여 필요한 시점에 자동으로 실행(인스턴스화)시킵니다.

2. Manifest에 반드시 등록되어야 한다
4대 컴포넌트는 모두 AndroidManifest.xml 파일에 등록해야 시스템이 해당 앱에 어떤 핵심 실행 단위가 있는지 인식할 수 있습니다.
(주의: Manifest에 등록되지 않은 4대 컴포넌트는 시스템이 인식하지 못하며, 실행되지 않습니다.)

3. 기타 요소와의 차이점
Fragment, View, Intent 등은 4대 컴포넌트의 기능을 보조하거나 구성하는 요소일 뿐, 시스템이 직접 실행 및 생명주기를 관리하는 독립적인 실행 단위가 아닙니다.


---

| 요소                    | 시스템이 직접 실행? | 생명주기 관리 주체           | Manifest 등록 필요 | 설명               |
| --------------------- | ----------- | -------------------- | -------------- | ---------------- |
| **Activity**          | ✅           | 시스템                  | ✅              | 사용자 화면(UI) 단위    |
| **Service**           | ✅           | 시스템                  | ✅              | 백그라운드 로직 실행      |
| **BroadcastReceiver** | ✅           | 시스템                  | ✅              | 브로드캐스트 수신        |
| **ContentProvider**   | ✅           | 시스템                  | ✅              | 앱 간 데이터 공유       |
| **Fragment**          | ❌           | Activity 내부          | ❌              | Activity의 일부로 동작 |
| **View**              | ❌           | Activity/Fragment 내부 | ❌              | 화면을 구성하는 UI 요소   |
| **Intent**            | ❌           | 없음                   | ❌              | 컴포넌트 간 메시지 전달 객체 |
| **Application**       | ❌           | 시스템이 1회 생성 (자동 등록)   | ❌              | 앱 전역 상태 관리 클래스   |
